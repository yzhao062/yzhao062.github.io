anywhere-agents

One config to rule all your AI agents: portable, effective, safer.

anywhere-agents is a maintained, opinionated configuration that follows your AI coding agents across every project, every machine, and every session. It ships shared skills, session hooks, and a small bootstrap script so behavior stays portable (it works the same in every repository), effective (curated writing rules, task routing, and skills), and safer (a PreToolUse guard blocks destructive Git and GitHub commands). Supports Claude Code and Codex today, with plans to grow.

I built and maintain this project because my own daily work spans many repositories and multiple agents, and every project was repeating the same setup: which skills to install, which hooks to wire up, which MCP servers to register, and which safety defaults to apply. anywhere-agents packages that setup into a single reusable configuration that stays fresh on every session.

This project sits across our Developer Tools for AI and AI Safety & Security directions, through session-start checks, a PreToolUse guard for destructive commands, and routing of review, figure, and writing tasks to the right skill.

Why this project

AI coding agents are now used across many repositories by the same person or team. Without a shared configuration, preferences drift: per-repo CLAUDE.md files fall out of sync, copy-pasted settings diverge on every tweak, and safety defaults live only in people's heads and have to be re-explained in every session. anywhere-agents publishes one curated configuration that any project can inherit in two lines of setup. The maintainer improves one file; every consuming repository picks it up on the next session.

Core capabilities

  • Idempotent bootstrap (PowerShell and Bash) that refreshes shared defaults, skills, and settings on every session.
  • Shared skills for review (implement-review), routing (my-router), figure building (ci-mockup-figure), and README polish (readme-polish), with pointer commands for Claude Code.
  • Session-start banner with environment checks (OS, agent, skills, hooks) and a GitHub Actions version audit.
  • PreToolUse guard (guard.py) that refuses to silently run destructive Git and GitHub commands such as push --force, reset --hard, and gh pr merge.
  • SessionStart hook (session_bootstrap.py) that refreshes shared config before any work begins.
  • Codex MCP integration and recommended Codex defaults (fast inference tier, high reasoning effort).
  • Cross-agent writing defaults: banned AI-tell words, formatting rules, and sentence-variety guidance.

Quick start

Install and bootstrap in any project root:

pipx run anywhere-agents   # Python path (zero-install with pipx)
npx anywhere-agents        # Node.js path (zero-install with Node 14+)

After bootstrap, the next session of Claude Code or Codex reads the shared AGENTS.md, inherits all defaults, and picks up future upstream changes automatically through git pull. Per-project overrides go in AGENTS.local.md, which bootstrap never overwrites.

Fork and customize

Everything is plain Markdown, Python, and JSON. Fork the repository, swap in your own skills or writing rules, and keep pulling upstream updates with git merge upstream/main. Groups and teams can publish their own shared configuration the same way.
